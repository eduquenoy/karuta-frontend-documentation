Dans Karuta un portfolio est représenté par un arbre structuré (voir STRUCTURES).

Il y a plusieurs types d'arbres dans Karuta.

  • modèle de portfolio
  • instance de portfolios
  • référentiel
  • rapport
  • batch
  • modèle de formulaire batch

Modèle de portfolio

Un modèle de portfolio est une structure d'arbre qui organise des ressources. Les ressources disponibles sont des textes courts, des textes longs, des images, des sons, des documents, des vidéos, etc.

Un modèle de portfolio est ensuite instancié pour chaque usager, c'est à dire que l'on fait une copie du modèle et cet arbre devientle portfolio spécifique à un usager.

Instance de portfolio

une instance de portfolio est une copie du modèle mais au moment de l'instantiation (copie) les rôles et droits sont créées pour chacun des éléements du portfolio.

Référentiel

Un référentiel est une liste d'éléments qui seront référencés dans les portfolios, par exemple, un référentiel de compétences.

Rapport

Un rapport est un script contenant des instructions spécifiques qui permettent de construire des tableaux de bord. Les instrcutions permettent de naviguer dans les portfolios et d'en afficher des éléments.

Batch

Un batch est un script contenant des commandes qui permettent d'une manière automatique de créer des usagers, des portfolios, de mettre à jour des éléments du portfolio, etc. Un fichier batch est généralement associé à un fichier CSV contenant, par exemple, la liste des usagers. L'éécution du batch créera alors automatiquement tous les portfolios des usagers. Un fichier batch peut être associé également à un formulaire.

Formulaire batch

Un formulaire batch permet d'exécuter un batch d'une manière unitaire pour un seul usager.

Principe de fonctionnement

Un portfolio ou arbre dans Karuta est construit en y ajoutant des branches un peu comme dans un Lego. Certaines branches sont prédéfinies : les élémnets de base de Karuta. D'autres branches sont construites par le desginer en assemblant des branches de base.

On peut ainsi construire des portfolios complexes, définir des menus permettant à l'usage final d'ajouter des branches dans son portfolio.

Modèle et instance

On distingue 2 types de portfolios:

  • Portfolio modèle - Construit par le designer. Les rôles associés sont : designer et all
  • Portfolio instance - Instancié à partir d'un modèle pour chaque utilisateur. Les rôles associés sont ceux définis par le designer dans le modèle

Le modèle de sécurité

  • Un portfolio a des rôles associés.
  • Un rôle a des droits associés (lire, écrire, soumettre, etc.) sur chacun des éléments du portfolio.
  • Le partage d'un portfolio se fait en indiquant un usager et un rôle.

Simulation des rôles par le designer

Dans un portfolio modèle, le designer peut simuler les différents rôles.
