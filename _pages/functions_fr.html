Karuta offre plusieurs fonctions préprogrammées qui peuvent être utilisées dans la métadonnées Menu.

  • Get_Multiple
  • Get_Get_Multiple
  • Import_Multiple
  • Import_GGMultiple
  • MoveTO
  • ExecReport_BatchCSV

Get_Multiple

La fonction Get_Multiple permet à l'utilisateur de sélectionner plusieurs éléments dans une liste. Pour chaque élément sélectionné, une composante réutilisable (Page ou Sous-section) est ajoutée au portfolio. Un élément de cette composante est alors mis à jour avec le code et le libellé de l'élément sélectionné.

Format

function,get_multiple/recherche/code_portfolio.tag_sémantique /tag_sémantique_a_mettre jour,libellé@fr,rôles

  • function : Mot réservé.
  • get_multiple : Mot réservé.
  • recherche : Recherche du Get_Resource (voir ci-dessus).
  • code_portfolio.tag_sémantique : Composante réutilisable à ajouter.
  • tag_sémantique_a_mettre_a_jour : Tag sémantique de l'élément de la composante à mettre à jour avec le code et le libellé sélectionné.
  • libellé@fr : Libellé du menu.
  • rôles : Rôle(s) voyant le menu.

Get_Get_Multiple

Équivalent à la fonction précédente, sauf que les choix aux utilisateurs sont restreints par la sélection dans un élément parent (même principe que pour les éléments Get_Get_Resource).

Format

function,get_get_multiple/recherche/ code_portfolio.tag_sémantique/tag_sémantique_à_mettre_à_jour,libellé@fr ,rôles

  • function : Mot réservé.
  • get_get_multiple : Mot réservé.
  • recherche : Recherche du Get_Get_Resource (voir ci-dessus).
  • code_portfolio.tag_sémantique : Composante réutilisable à ajouter.
  • tag_sémantique_à_mettre_à_jour : Tag sémantique de l'élément de la composante à mettre à jour avec le code et le libellé sélectionné.
  • libellé@fr : Libellé du menu.
  • rôles : Rôle(s) voyant le menu.

Import_Multiple

La fonction Import_Multiple permet à l'utilisateur d'ajouter plusieurs éléments à partir d'une liste. Comme les commandes précédentes, un query permet de sélectionner les éléments de la liste. Mais avec Import_Multiple, les codes des éléments sélectionnés sont utilisés pour importer les éléments ayant ce code dans le portfolio d'importation. Le code du portfolio d'importation peut être le même que celui de recherche, dans ce cas on importe les éléments recherchés et cochés. Mais le code du portfolio d'importation peut être différent de celui de recherche, dans ce cas on importe les éléments ayant le même code que ceux cochés. Le portfolio d'importation est le portfolio dans lequel on va chercher les éléments à importer.

OPTION : Le code du portfolio d'importation peut prendre la forme ?. tag_sémantique. Dans ce cas le code du portfolio sera le code de l'élément coché et le tag sémantique de l'élément importé sera tag_sémantique.

ATTENTION : Si les éléments à importer sont des items, le code de l'item (Item - Code) doit être répété dans le code du noeud (Code).

Format

function,import_multiple/recherche/code_portfolio, libellé@fr,rôles

  • function : Mot réservé.
  • import_multiple : Mot réservé.
  • recherche : Recherche du Get_Resource (voir "recherche" ci-dessus) — retourne une liste de codes.
  • code_portfolio : Code du portfolio duquel on importe le noeud avec le(s) code(s) sélectionné(s) — peut être ou pas le même que celui du Get_Resource.
    OPTION : ?.tag_sémantique
  • libellé@fr : Libellé du menu.
  • rôles : Rôle(s) voyant le menu.

Menu - Import_GGMultiple

La fonction Import_GGMultiple permet à l'utilisateur d'ajouter plusieurs éléments à partir d'une liste. Comme les commandes précédentes, une recherche permet de sélectionner les éléments de la liste. Mais avec Import_GGMultiple, les codes des éléments sélectionnés sont utilisés pour importer les éléments ayant ce code dans le portfolio d'importation. Le code du portfolio d'importation peut être le même que celui de recherche, dans ce cas on importe les éléments recherchés et cochés. Mais le code du portfolio d'importation peut être différent de celui de recherche, dans ce cas on importe les éléments ayant le même code que ceux cochés.

OPTION 1 : Dans la recherche le code du portfolio peut être ?. Dans ce cas le code du portfolio de recherche sera le code de l'élément parent.

OPTION 2 : Le code du portfolio d'importation peut prendre la forme ?. Dans ce cas le code du portfolio d'importation sera le code de l'élément parent trouvé.

Format

function,import_ggmultiple/recherche/code_portfolio, libellé@fr,rôles

  • function : Mot réservé.
  • import_ggmultiple : Mot réservé.
  • recherche : Recherche du Get_Get_Resource (voir "recherche" ci-dessus) — retourne une liste de codes.
    OPTION 1 : Le code du portfolio peut être le mot réservé ?.
  • code_portfolio : Code du portfolio duquel on importe le noeud avec le(s) code(s) sélectionné(s).
    OPTION 2 : Le code du portfolio peut être le mot réservé ?.
  • libellé@fr : Libellé du menu.
  • rôles : Rôle(s) voyant le menu.

Exemple d'utilisation : Un Get-Resource permet de sélectionner un référentiel parmi une liste de référentiels (le code de l'élément Item doit être le code du portfolio du référentiel correspondant). Un Import_GGMultiple permet d'importer des éléments du référentiel sélectionné dans le Get_Resource.

Menu : function,import_ggmultiple/?.child.referentiel.skill$.label /?,Ajouter une compétence@fr,auteur

Remarque : Ce menu se trouve sur un élément Sous-section et le Get_Resource est à l'intérieur (d'où l'utiliation du mot résevé child. referentiel est à la fois le tag sémantique du Get_Resource et le tag sémantique que des éléments Item de la liste de référentiels. Chaque Item a pour code le code du portfolio référentiel correspondant. Les points d'interrogation ? font référence à ce code de portfolio référentiel.
