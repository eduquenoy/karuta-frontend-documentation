KARUTA est un logiciel open source permettant de construire des portfolios. C'est un outil flexible qui permet de construire aussi bien des portfolios de présentation, que des portfolios d'apprentissage ou encore des portfolios d'évaluation.

Le développement de portfolios peut se faire à partir de templates fournis par la communauté et qui seront ensuite adaptés aux besoins spécifiques, ou encore à partir des éléments de base fournis par Karuta.

La documentation de Karuta est organisée en trois sections :

  • Portfolio
  • Rapport
  • Batch

La section Portfolio présente les éléments de structure, les ressources (les éléments qui vont contenir les données), les métadonnées et la mise en forme (CSS).

La section Rapport présente les commandes permettant de construire des tableaux de bord, la mise en forme des rapports ainsi que les tableaux de bord qui sont éditables par les usagers. Les commandes d'un script de rapport permettent de sélectionner des éléments de portfolios et de les présenter sous forme de tableau.

La section Batch présente les commandes des scripts batch. Ces commandes permettent d'automatiser la création, modification et suppression de portfolios ainsi que la création ou suppresion d'usagers.

Remarques d'ordre général

KARUTA est multilingue (français et anglais par défaut). Les libellés des éléments de portfolios sont écrits chacun dans la langue sélectionnée dans le menu. Les libellés dans les menus et dans l'aide sont suffixés par le code de la langue et séparés par une barre oblique /. Exemple : Ajouter une photo@fr/Add a photo@en

Un usager de Karuta est associé à un 'rôle' dans un portfolio. Par exemple, le rôle d'étudiant ou de professeur. Pour chaque portfolio, un groupe d'usagers est créé pour chaque rôle rencontré dans le portfolio. Ensuite, pour associer un rôle à un usager, on place cet usager dans le groupe du rôle correspondant.

Il y a trois rôles prédéfinis :

  • designer C'est l'usager qui construit les portfolios. Il a tous les droits sur les éléments du portfolio, indépendamment des divers rôles spécifiés dans les métadonnées d'un élément.
  • all Ce mot-clé inclus tous les rôles définis sur le portfolio.
  • user Ce mot-clé sera remplacé par l'identifiant de l'usager lors de l'instantiation du portfolio. Cela devient donc un rôle unique puisqu'il n'y a qu'un seul usager qui peut avoir ce rôle.
