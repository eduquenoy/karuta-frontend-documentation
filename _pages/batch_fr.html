Fonctionnement

Ensemble de commandes de script permettant d'automatiser la création, modification et suppression de portfolios, ainsi que la création ou suppression d'usagers.

À partir d'un ou plusieurs modèles de portfolios, d'un script batch et d'un fichier CSV, KARUTA crée les usagers et leurs portfolios. Le fichier CSV contient les données relatives à chaque usager à créer. Le script de batch indique les opérations à effectuer : créer l'usager, créer son ou ses instances de portfolios à partir de modèles de portfolio, partager les nouveaux portfolios avec les usagers dans des rôles spécifiés, remplir les données de l'usager dans les portfolios, etc.

Exemple de script batch

Ce script correspond à l'algorithme suivant :

Pour chaque ligne du fichier CSV

  • Créer l'usager si il n'existe pas
  • Créer un portfolio Profil
  • Partager le portfolio Profil avec l'usager dans le rôle student
  • Mettre à jour les champs Nom et Prénom dans le portfolio Profil
  • Créer un portfolio Évaluation
  • Partager le portfolio Évaluation avec l'usager dans le rôle student
  • Mettre à jour le Proxy du portfolio Évaluation vers le portfolio Profil

Exemple de fichier CSV

Le fichier CSV contient les informations nécessaires à l'exécution du batch.

Format du fichier CSV :

  • Ligne 1 : Les noms des variables globales. Les variables globales sont les variables qui ont les même valeurs pour chaque usager à créer : par exemple, les codes des modèles de portfolio à instancier, un code de programme ou une session.
    model_code est un mot réservé : c'est le code du portfolio contenant le script de batch à exécuter.
    Dans le script de batch, les références aux variables globales sont précédées de //.
  • Ligne 2 : Les valeurs des variables globales. Dans l'exemple présenté la valeur de model_code est createStudents : c'est le code du portfolio contenant le script de batch.
  • Ligne 3 : Les noms des variables locales. Les variables locales sont les variables qui ont des valeurs différentes pour chaque usager à créer : par exemple, le nom de l'étudiant, son prénom, courriel, etc.
  • Lignes 4 et + : Les valeurs des variables locales, habituellement un usager par ligne.

Les commandes

Commande d'exécution :

  • ForEachLine

Commandes pour les usagers :

  • CreatePerson
  • DeletePerson
  • JoinUserGroup
  • LeaveUserGroup

Commandes pour les portfolios :

  • CreateTree
  • ShareTree
  • UnshareTree
  • SelectTree
  • DeleteTree

Commande pour les groupes de portfolios :

  • JoinPortfolioGroup

Commandes pour les groupes d'usagers :

  • ShareUserGroup
  • UnshareUserGroup

Commandes pour ajout dans un portfolio :

  • ImportNode
  • MoveUp

Commandes de mise à jour d'éléments de portfolio :

  • UpdateField
  • UpdateProxy
  • UpdateDashboard
  • UpdateTreeRoot

Commandes de mise à jour de métadonnées :

  • UpdateMetadataQuery
  • UpdateMetadataMenu
  • UpdateMetadataInline
  • UpdateMetadata
  • UpdateMetadataWad

ForEachLine

ForEachLine est une boucle et généralement le point de départ d'un script de batch. Les commandes à l'intérieur de l'élément ForEachLine seront exécutées pour chaque ligne du fichier CSV (lignes 4+).

CreatePerson

Créer un usager. Ne pas oublier de remplacer les valeurs par défaut de cet élément par vos propres noms de variables locales (ligne 3 du fichier CSV).

Champs :

  • Identifiant : Identifiant unique du l'usager qui servira de login dans le système Karuta (par exemple un matricule ou courriel).
  • Prénom : Prénom de l'usager.
  • Nom : Nom de famille de l'usager.
  • Courriel : Courriel de l'usager.
  • Mot de passe : Le mot de passe de l'usager dans le système Karuta.
  • Créateur? : Le compte utilisateur créé doit-il avoir les droits d'un designer Karuta? (En général, non!)

DeletePerson

Supprimer un usager. (Attention : supprimer un usager ne supprime pas ses portfolios!)

Champ :

  • Identifiant : Identifiant unique de l'usager à supprimer (son login). Ajouter un élément select pour puiser l'identifiant dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement l'identifiant de l'usager.

JoinUserGroup

Ajouter un usager à un groupe d'usagers.

Champs :

  • Personne : Identifiant de l'usager (son login). Ajouter un élément select pour puiser l'identifiant dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement l'identifiant de l'usager.
  • Groupe : Nom du groupe d'usagers (exemple : enseignants2017). Ajouter un élément select pour puiser le nom dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement le nom du groupe d'usagers.

LeaveUserGroup

Retirer un usager d'un groupe d'usagers.

Champs :

  • Personne : Identifiant de l'usager (son login). Ajouter un élément select pour puiser l'identifiant dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement l'identifiant de l'usager.
  • Groupe : Nom du groupe d'usagers (exemple : enseignants2017). Ajouter un élément select pour puiser le nom dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement le nom du groupe d'usagers.

CreateTree

Créer une instance de portfolio à partir d'un modèle de portfolio.

Champs :

  • ID  Attribuer un ID qui sera utilisé dans des commandes subséquentes pour faire référence à ce portfolio (par exemple pour le partager avec un usager).
  • Modèle  Code du modèle de portfolio à instancier. Format : code du projet.code du portfolio modèle. Ajouter des éléments select pour des variables globales ou locales du fichier CSV et/ou des éléments value pour spécifier directement des valeurs.
  • Code  Code du portfolio qui sera créé. Format : code du projet.code du portfolio. Ajouter des éléments select pour des variables globales ou locales du fichier CSV et/ou des éléments value pour spécifier directement des valeurs.
  • Libellé  Libellé du portfolio qui sera créé. Ajouter des éléments select pour des variables globales ou locales du fichier CSV et/ou des éléments value pour spécifier directement des valeurs.

ShareTree

Partager un portfolio avec un usager dans un rôle spécifié.

Champs :

  • Sélection arbre : Sélectionner l'ID du portfolio concerné par le partage. L'ID a précédement été attribué soit dans un élément CreateTree ou dans un élément SelectTree.
  • Personne : Identifiant de l'usager. Ajouter un élément select pour puiser l'identifiant dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement l'identifiant de l'usager.
  • Rôle : Rôle de l'usager dans le portfolio (exemple : etudiant). Ajouter un élément select pour puiser le rôle dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement le rôle de l'usager.

SelectTree

Sélectionner un portfolio qui sera ensuite référencé dans d'autres commandes du script.

Champs :

  • Code : Code du portfolio à sélectionner. Format : code du projet.code du portfolio. Ajouter des éléments select pour des variables globales ou locales du fichier CSV et/ou des éléments value pour spécifier directement des valeurs.

DeleteTree

Supprimer un portfolio.

Champs :

  • Code : Code du portfolio à supprimer. Format : code du projet.code du portfolio. Ajouter des éléments select pour des variables globales ou locales du fichier CSV et/ou des éléments value pour spécifier directement des valeurs.

JoinPortfolioGroup

Ajouter un portfolio à un groupe de portfolios.

Champs :

  • Sélection arbre : Sélectionner l'ID du portfolio concerné par le partage. L'ID a précédement été attribué soit dans un élément CreateTree ou dans un élément SelectTree.
  • Groupe : Nom du groupe de portfolios (exemple : cohorte2017). Ajouter un élément select pour puiser le nom dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement le nom du groupe de portfolios.

ShareUserGroup

Partager un portfolio avec un groupe d'usagers dans un rôle spécifié.

Champs :

  • Sélection arbre : Sélectionner l'ID du portfolio concerné par le partage. L'ID a précédement été attribué soit dans un élément CreateTree ou dans un élément SelectTree.
  • Groupe : Nom du groupe d'usagers. Ajouter un élément select pour puiser le nom dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement le nom du groupe d'usagers.
  • Rôle : Rôle des usagers dans le portfolio. Ajouter un élément select pour puiser le rôle dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement le rôle des usagers.

UnshareUserGroup

Départager un portfolio d'un groupe d'usagers dans un rôle spécifié.

Champs :

  • Sélection arbre : Sélectionner l'ID du portfolio concerné par le partage. L'ID a précédement été attribué soit dans un élément CreateTree ou dans un élément SelectTree.
  • Groupe : Nom du groupe d'usagers. Ajouter un élément select pour puiser le nom dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement le nom du groupe d'usagers.
  • Rôle : Rôle des usagers dans le portfolio. Ajouter un élément select pour puiser le rôle dans une variable globale ou locale du fichier CSV, ou un élément value pour spécifier directement le rôle des usagers.

UpdateField

Mettre à jour un champ de type texte court dans une instance de portfolio.

Champs :

  • Sélection : Ressource à mettre à jour. Format : ID.tag_sémantique. L'ID du portfolio a précédement été attribué soit dans un élément CreateTree ou dans un élément SelectTree, et le tag sémantique est est celui de la ressource à mettre à jour dans le portfolio.
  • Valeur du champ : Valeur à mettre à jour. Ajouter un élément select pour puiser la valeur dans une variable globale ou locale du fichier CSV (exemple : nomEtudiant), ou un élément value pour spécifier directement la valeur du champ.

ImportNode

Importer un élément dans un portfolio.

Champs :

  • Sélection destination : Élément de structure où importer le noeud. Format : ID.tag_sémantique. L'ID du portfolio a précédement été attribué soit dans un élément CreateTree ou dans un élément SelectTree, et le tag sémantique est est celui de l'élément de structure où importer le noeud.
  • Sélection source : Élément à importer dans le portfolio. Format : code_portfolio.tag_sémantique.

MoveUp

La commande permet d'importer un ou des éléments dans un portfolio.

Champ :

Sélection C'est le noeud que l'on veut déplacer. Format : ID.tag_sémantique. ID est une référence à un portfolio créé ou sélectionné précedemment.

UpdateProxy

La commande permet de mettre à jour un proxy.

Champs :

Sélection proxy C'est le noeud proxy que l'on veut mettre à jour. Format : ID.tag_sémantique. ID est une référence à un portfolio créé ou sélectionné précedemment.

Sélection source C'est le noeud vers lequel pointe le proxy. Format : code_portfolio.tag_sémantique.

UpdateDashboard

La commande permet de mettre à jour l'uuid d'un tableau de bord.

Champs :

Sélection C'est le noeud proxy que l'on veut mettre à jour. Format : ID.tag_sémantique. ID est une référence à un portfolio créé ou sélectionné précedemment.

Dashboard uuid Cela peut-être soit une valeur (value) et/ou une variable (select) soit une référence à un noeud source (SourceSelect - Format : ID.UUDI où ID est une réfrence à un portfolio déjà créé ou sélectionné et UUID est un mot réservé. La valeur est alors l'uuid du porfolio ID)

UpdateTreeRoot

La commande permet de mettre à jour le code et le libellé d'un portfolio.

Champs :

Ancien code Il faut ajouter un select pour une variable et/ou value pour spécifier directement une valeur

Nouveau code Il faut ajouter un select pour une variable et/ou value pour spécifier directement une valeur

Libellé Il faut ajouter un select pour une variable et/ou value pour spécifier directement une valeur

UpdateMetadataQuery

La commande permet de mettre à jour l'attribut recherche (query) d'un noeud.

Champs :

Sélection C'est le noeud que l'on veut mettre à jour. Format : ID.tag_sémantique. ID est une référence à un portfolio créé ou sélectionné précedemment.

Valeur query Il faut ajouter un select pour une variable et/ou value pour spécifier directement une valeur

UpdateMetadataMenu

La commande permet de mettre à jour l'attribut menu d'un noeud.

Champs :

Sélection C'est le noeud que l'on veut mettre à jour. Format : ID.tag_sémantique. ID est une référence à un portfolio créé ou sélectionné précedemment.

Valeur menu Il faut ajouter un select pour une variable et/ou value pour spécifier directement une valeur

UpdateMetadataInline

La commande permet de mettre à jour l'attribut édition en ligne (inline) d'un noeud.

Champs :

Sélection C'est le noeud que l'on veut mettre à jour. Format : ID.tag_sémantique. ID est une référence à un portfolio créé ou sélectionné précedemment.

Valeur inline (Y ou N) Il faut ajouter un select pour une variable et/ou value pour spécifier directement une valeur

UpdateMetadata

La commande est plus générale que les précédentes et permet de mettre à jour n'importe quel attribut de metadata d'un noeud.

Champs :

Sélection C'est le noeud que l'on veut mettre à jour. Format : ID.tag_sémantique. ID est une référence à un portfolio créé ou sélectionné précedemment.

attribut C'est le nom de l'attibut à mettre à jour.

Valeur atttribut Il faut ajouter un select pour une variable et/ou value pour spécifier directement une valeur

UpdateMetadataWad

La commande est plus générale que les précédentes et permet de mettre à jour n'importe quel attribut de metadataWad d'un noeud.

Champs :

Sélection C'est le noeud que l'on veut mettre à jour. Format : ID.tag_sémantique. ID est une référence à un portfolio créé ou sélectionné précedemment.

attribut C'est le nom de l'attibut à mettre à jour.

Valeur atttribut Il faut ajouter un select pour une variable et/ou value pour spécifier directement une valeur
