Les métadonnées des éléments permettent de leur attribuer des identifiants uniques et aussi d'en spécifier le comportement. Par exemple, dans le cas d'éléments structurels, on peut définir des menus qui permettent l'ajout d'éléments supplémentaires par le designer ou même par l'utilisateur final. Dans le cas des éléments de ressource, on peut définir les rôles attribués à chaque droit (qui peut faire quoi sur l'élément). C'est aussi dans les métadonnées qu'on spécifie quels sont les éléments qui vont remplir une boîte de choix. Finalement on peut aussi spécifier dans les métadonnées quels utilisateurs pourront partager leur portfolio avec une tierce partie et sous quelles conditions de sécurité.

On accède aux métadonnées d'un élément via sa boîte d'édition.

Quelque unes des métadonnées d'un élément Karuta :

  • Tag sémantique
  • Code
  • Noeud multilingue
  • Ressource multilingue
  • Encrypter la ressource
  • Rôles - Voir
  • Rôles - Supprimer
  • Rôles - Éditer ressource
  • Rôles - Commenter
  • Rôles - Soumettre

Tag sémantique

Le tag sémantique d'un élément exprime sa sémantique. C'est aussi le moyen de rechercher des éléments dans un portfolio. Le tag sémantique est utilisé dans le champ 'recherche' d'un élément (voir ci-dessous) ainsi que dans les scripts de rapport et de batch.

ATTENTION : la recherche de tag sémantique se fait par inclusion. Par exemple, si on recherche les éléments de tag sémantique 'competence', les éléments avec le tag sémantique 'sous-competence' seront également trouvés, car le mot 'competence' est inclus dans le mot 'sous-competence'.

Code

Le code d'un élément de portfolio doit être unique dans ce portfolio. Il est utilisé dans les listes déroulantes (élément Get_Resource, élément Get_Get_Resource, menu Get_Multiple, menu Get_Get_Multiple, menu Import_Multiple) pour identifier la sélection de l'usager.

Par défaut le code et le libellé de l'élément apparaissent dans la liste déroulante et seul le libellé de l'option sélectionnée par l'usager apparaît ensuite dans l'élément ressource. Ce comportement peut être modifié par l'ajout d'un ou plusieurs des symboles suivants au code de l'élément :

  • @ rendra le code non visible dans la liste déroulante
  • # rendra le code visible dans la ressource
  • & rendra la valeur associée à l'élément visible dans la ressource (uniquement pour l'élément Item)
  • % rendra le libellé non visible dans la ressource
  • $ rendra le libellé non visible dans la liste déroulante

Noeud multilingue

L'attribut 'noeud multilingue' d'un élément permet de spécifier si le libellé du noeud prend des valeurs différentes ou non dans le cas où le portfolio est multilingue. Si le noeud n'est pas multilingue, la valeur du libellé sera stockée dans la langue par défaut définie dans le fichier de configuration _init.js.

Remarque : le code n'est jamais multilingue. Il a la même valeur quelle que soit la langue sélectionnée.

Ressource multilingue

L'attribut 'ressource multilingue' d'un élément permet de spécifier si les données entrées pour la ressource prennent des valeurs différentes ou non dans le cas où le portfolio est multilingue. Si la ressource n'est pas multilingue les valeurs entrées pour l'élément seront stockées dans la langue par défaut définie dans le fichier de configuration _init.js.
Par exemple, un nom ou une date de naissance ne sont habituellement pas dépendants de la langue, alors qu'un texte est définitivement différent dans chaque langue.

Encrypter la ressource

Cet attribut permet de spécifier si la ressource est encryptée. Dans le cas positif, à l'ouverture du portfolio, une phrase-clé est demandée à l'usager. Cette phrase-clé sert à encoder/décoder les valeurs de la ressource. Cette phrase-clé n'est pas stockée dans Karuta et doit donc être mémorisée par l'usager, car elle lui est demandée à chaque ouverture du portfolio.

Rôles - Voir

La liste des rôles, séparés par un espace, pouvant voir l'élément. Exemple : etudiant professeur
Par défaut le droit de voir est attribué au rôle all. Il s'agit de l'unique droit à avoir une valeur par défaut.

Rôles - Supprimer

La liste des rôles, séparés par un espace, pouvant supprimer l'élément.

Rôles - Éditer ressource

La liste des rôles, séparés par un espace, pouvant éditer la ressource (y stocker des données). Disponible uniquement sur les éléments ressource, non disponible sur les éléments de structure.

Rôles - Commenter

La liste des rôles, séparés par un espace, pouvant commenter l'élément. Ce droit ajoute un éditeur de texte à la ressource; les commentaires sont datés et signés du nom de l'usager qui l'a écrit. Pour pouvoir commenter un élément, l'usager doit avoir également le droit d'éditer la ressource.

Rôles - Soumettre

La liste des rôles, séparés par un espace, pouvant soumettre l'élément. Ce droit ajoute un bouton 'soumettre'. Lorsque l'usager soumet un élément, les droits d'édition et/ou de suppression sont retirés de l'élément et de tous ses enfants.

Recherche

Le champ 'recherche' est utilisé par :

  • les listes déroulantes (éléments Get_Resource et Get_Get_Resource)
  • les menus aux usagers sur les Sections, les Pages et les Sous-sections
  • les listes de choix-multiple implémentées à travers des menus (menu Get_Multiple, menu Get_Get_Multiple et menu Import_Multiple)

Format du Get_ (Get_Resource ou Get_Multiple)

code_portfolio.tag_sémantique.label

  • code_portfolio : Code du portfolio référentiel où faire la recherche. Le code du portfolio est formé de deux parties séparées par un point : la première partie est le code du projet dans lequel se trouve le portfolio référentiel, la deuxième partie est le code du portfolio référentiel lui-même. Le code du projet est optionel : si il est absent, il est remplacé par le code du projet dans lequel se trouve le portfolio courant (le portfolio qui contient le Get_).
  • tag_sémantique : Tag sémantique des élements à chercher.
  • label : Mot réservé.

Format Get_Get_ (Get_Get_Resource ou Get_Get_Multiple)

code_portfolio.position.tag_sémantique1.tag_sémantique2.label

  • code_portfolio : Code du portfolio référentiel où faire la recherche.
  • position : Indique où chercher dans le portfolio courant le code de l'élément parent qui va permettre de restreindre la recherche dans le référentiel. Prend une des valeurs suivantes : child, sibling, parent, parent.parent, parent.parent.parent
  • tag_sémantique1 : Tag sémantique du parent qui restreint la recherche.
  • tag_sémantique2 : Tag sémantique des éléments à chercher dans le référentiel.
  • label : Mot réservé.

Menu - Get_Multiple

La fonction Get_Multiple permet à l'usager d'ajouter plusieurs éléments à partir d'une liste. Un query permet de sélectionner les éléments de la liste. L'usager peut cocher un ou plusieurs items de la liste. Pour chaque item de la liste, Karuta va ajouter un élément prédéfini contenant au moins un Get-Resource qui sera mis à jour avec la sélection de l'usager.

Format

function,get_multiple/query/code_portfolio.tag_sémantique/get_resource_tag_sémantique,libellé@fr,rôles

  • function : Mot réservé.
  • get_multiple : Mot réservé.
  • query : Recherche du Get_Resource.
  • code_portfolio.tag_sémantique : Élément à ajouter (peut être juste le Get-Resource).
  • get_resource_tag_sémantique : Tag sémantique du Get_Resource (dans l'élément à ajouter).
  • libellé@fr : Libellé du menu.
  • rôles : Rôle(s) voyant le menu.

Menu - Get_Get_Multiple

La fonction Get_Get_Multiple permet à l'usager d'ajouter plusieurs éléments à partir d'une liste. Comme pour le Get_Multiple, un query permet de sélectionner les éléments de la liste. L'usager peut cocher un ou plusieurs items de la liste. Pour chaque item de la liste, Karuta va ajouter un élément prédéfini contenant au moins un Get-Resource qui sera mis à jour avec la sélection de l'usager.

Format

function,get_get_multiple/query/code_portfolio.tag_sémantique/get_resource_tag_sémantique,libellé@fr, rôles

  • function : Mot réservé.
  • get_get_multiple : Mot réservé.
  • query : Recherche du Get_Get_Resource.
  • code_portfolio.tag_sémantique : Composante réutilisable à ajouter (peut être juste le Get-Resource).
  • get_resource_tag_sémantique : Tag sémantique du Get_Resource (dans la composante à ajouter).
  • libellé@fr : Libellé du menu.
  • rôles : Rôle(s) voyant le menu.

Menu - Import_Multiple

La fonction Import_Multiple permet à l'usager d'ajouter plusieurs éléments à partir d'une liste. Comme les commandes précédentes, un query permet de sélectionner les éléments de la liste. Mais avec Import_Multiple, les codes des éléments sélectionnés sont utilisés pour importer les éléments ayant ce code dans le portfolio d'importation. Le code du portfolio d'importation peut être le même que celui de recherche, dans ce cas on importe les éléments recherchés et cochés. Mais le code du portfolio d'importation peut être différent de celui de recherche, dans ce cas on importe les éléments ayant le même code que ceux cochés. Le portfolio d'importation est le portfolio dans lequel on va chercher les éléments à importer.

OPTION : le code du portfolio d'importation peut être de la forme ?. semantictag. Dans ce cas le code du portfolio sera le code de l'élément coché et le tag sémantique de l'élément importé sera tag_sémantique.

Format

function,import_multiple/query/code_portfolio,libellé@fr,rôles

  • function : Mot réservé.
  • import_multiple : Mot réservé.
  • query : Query du Get_Resource - retourne une liste de codes.
  • code_portfolio : Code du portfolio duquel on importe le noeud avec le(s) code(s) sélectionné(s) - peut être ou pas le même que celui du Get_resource. OPTION ?.tag_sémantique.
  • libellé@fr : Libellé du menu.
  • rôles : Rôle(s) voyant le menu.

Menu - Import_GGMultiple

La fonction Import_GGMultiple permet à l'usager d'ajouter plusieurs éléments à partir d'une liste. Comme les commandes précédentes, un query permet de sèlectionner les éléments de la liste. Mais avec Import_GGMultiple, les codes des éléments sélectionnés sont utilisés pour importer les éléments ayant ce code dans le portfolio d'importation. Le code du portfolio d'importation peut être le même que celui de recherche, dans ce cas on importe les éléments recherchés et cochés. Mais le code du portfolio d'importation peut être différent de celui de recherche, dans ce cas on importe les éléments ayant le même code que ceux cochés.

OPTION1 : Dans le query le code du portfolio peut être ?. Dans ce cas le code du portfolio de recherche sera le code de l'élément parent.

OPTION2 : le code du portfolio d'importation peut être de la forme ?. Dans ce cas le code du portfolio d'importation sera le code de l'élément parent trouvé.

Format

function,import_ggmultiple/query/code_portfolio,libellé@fr,rôles

  • function : Mot réservé.
  • import_ggmultiple : Mot réservé.
  • query : Recherche du Get_Get_Resource - retourne une liste de codes. OPTION1 : code du portfolio peut-être le mot-réservé ?.
  • code_portfolio : Code du portfolio duquel on importe le noeud avec le(s) code(s) sélectionné(s). OPTION2 : code du portfolio peut-être le mot-réservé ?.
  • libellé@fr : Libellé du menu.
  • rôles : Rôle(s) voyant le menu.

Exemple d'utilisation : un Get-Resource permet de sélectionner un référentiel parmi une liste de référentiels (le code de l'item doit être le code du portfolio du référentiel). Un Import_GGMultiple permet d'importer des éléments du référentiel sélectionné dans le Get_resource.

menu : function,import_ggmultiple/?.child.referentiel.skill$.label/?,Ajouter une compétence@fr,auteur

Remarque: Ce menu se trouve sur une unitstructure et le Get_resource est en-dessous d'où le child, referentiel est le tag du Get_resource ainsi que des items de la liste de référentiels. Chaque item a pour code le code du portfolio référentiel. Les points d'interrogation ? font référence à ce code de portfolio référentiel

Partage par l'usager

La fonction Partage par l'usager permet à l'usager d'envoyer un courriel avec un hyperlien qui partage, en tout ou en partie, son portfolio avec un tiers.

Format

rôle

rôle,rôle destinataire,rôles et/ou courriels,niveau (0-4),durée de vie du lien (en heures),libellé du bouton@fr,condition (optionel)

  • rôle Rôle de l'usager qui peut faire le partage.
  • rôle destinataire Rôle qu'aura le destinataire dans le portfolio.
  • rôles et/ou courriels Rôles et/ou courriels des destinataires. Si c'est un rôle, tous les personnes ayant ce rôle dans le portfolio seront destinataire d'un courriel.
  • niveau (0-4) : Le niveau peut prendre les valeurs de 0 à 4 (voir ci-dessous).
  • libellé du bouton@fr : Libellé qui apparaitra à l'usager qui peut faire le partage.
  • condition Condition pour l'apparation du bouton de partage. Exemple : submitted=='Y'
    • Niveau 0 – utilisateur Karuta

    L’utilisateur existe et le portfolio est déjà partagé avec l’usager et le rôle. Le mot de passe est demandé si le destinataire n'est pas loggé.

    Niveau 1

    L’utilisateur existe et le portfolio est déjà partagé avec l’usager et le rôle. Le mot de passe n'est pas demandé. Si l'utilisateur n'est pas loggé il obtient un logging temporaire. (Un bouton submit devrait être toujours ajouté)

    Niveau 2

    L’utilisateur existe et le portfolio n’est pas partagé avec l’usager et le rôle. Le système partage le portfolio avec l’usager et le rôle. Si l'utilisateur n'est pas loggé il obtient un logging temporaire. (Un bouton submit devrait être toujours ajouté)

    Niveau 3

    Si l’utilisateur n’existe pas le système crée le compte. Le système partage le portfolio avec l’usager et le rôle. Si l'utilisateur n'est pas loggé il obtient un logging temporaire. (Un bouton submit devrait être toujours ajouté)

    Niveau 4 – Lien public

    Si l'utilisateur n'est pas loggé il obtient un logging temporaire. Il a automatiquement (loggé ou non) le rôle all comme uager public.

    Pour une documentation complète et des exemples d'utilisation consulter Doc Partage par l'usager
