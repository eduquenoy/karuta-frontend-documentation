Des portfolios complets, des parties de portfolios ou même des ressources spécifiques peuvent être partagées par un usager donné à l’aide d’un bouton. Les usagers qui recevront le partage en seront avisés par courriel (le courriel contient un hyperlien vers le portfolio partagé).

Ils pourront alors intervenir dans le portfolio selon le rôle qui leur sera assigné. Le partage permet deux possibilités :

  1. avertir un acteur du portfolio qu’il doit intervenir;
  2. permettre à un usager qui n’a pas actuellement les droits d’intervenir dans un portfolio de le faire.

Dans le portfolio

Ainsi, si dans son portfolio l’étudiant clique sur Partager avec un pair, le système envoie un courriel à un pair avec un hyperlien vers une section du portfolio dans laquelle le pair pourra inscrire son commentaire.

La spécification du partage se fait dans l’onglet métadonnées d’une section, d’une page, d’une sous-section ou même d’une ressource.

L’évaluateur externe reçoit un courriel et n’a qu’à cliquer sur l’hyperlien pour procéder à l’évaluation; il n’a pas à s’identifier dans le système Karuta. L’hyperlien est valide pendant 12 heures.

Cas 4 – L’étudiant demande à un de ses collègues d’inscrire un commentaire dans son portfolio (évaluation par les pairs); le collègue possède un compte Karuta.

Partager

rôle qui partage,rôle destinataire,rôles et/ou courriels,niveau (0-4),durée de vie du lien (en heures),button label@en/libellé du bouton@fr[,condition][;autre partage]

etudiant,pair,?,1,12,Demander un commentaire@fr

Le collègue reçoit un courriel et n’a qu’à cliquer sur l’hyperlien pour écrire son commentaire.

Note : Le symbole « ? » fait en sorte que lorsque l’étudiant cliquera sur Demander un commentaire, une fenêtre s’ouvrira pour lui permettre de spécifier le courriel de son collègue. On fait ici l’hypothèse que l’étudiant a un grand nombre de collègues et que le designer n’a pas spécifié de façon précise les collègues qui seront mobilisés, de façon à donner plus de latitude à l’étudiant dans le choix du pair qui pourra l’évaluer.

Cas généraux. Les deux tableaux ci-dessous expliquent et précisent la valeur des paramètres de la boîte de partage.

Rôle qui partage Habituellement l’étudiant mais tous les usagers qui ont un rôle défini dans le portfolio peuvent aussi partager.
Rôle du destinataire Le rôle qu’aura le destinataire dans le portfolio (par exemple « superviseur » – le destinataire pourra faire ce que peut faire un superviseur).
Rôles et/ou courriels
    Le rôle du destinataire dans le portfolio ou encore son courriel :
  • - Par exemple si « superviseur », le message de partage sera envoyé à la personne qui a le rôle superviseur dans le portfolio. L'usager n'entre pas à la main la personne avec qui partager, ceci est déterminé par le système.
  • - On peut aussi inscrire directement une adresse courriel si le destinataire est connu.
  • - Finalement on peut remplacer ce paramètre par « ? », ce qui permet à l’utilisateur qui partage de spécifier le courriel de la personne avec qui partager.
Niveau (0,4) Le niveau de sécurité détermine si on oblige le destinataire à se connecter à Karuta avec son identifiant et mot de passe avant de voir ce qui lui est partagé. Il permet aussi à des destinataires qui n’ont pas de compte dans Karuta d’éditer des ressources sans se connecter (niveau 3).
Durée de vie du lien (en heures) Pour des raisons de sécurité, le designer peut spécifier la durée de vie du lien qui sera envoyé.
button label@en/ libellé du bouton@fr Le libellé du menu Partager dans la langue appropriée. Ne pas oublier le / si bilingue.
Condition Il est possible de spécifier certaines conditions (facultatif).
Niveau Conditions
0 Le destinataire existe dans le système Karuta et le portfolio est déjà partagé avec le destinataire dans le bon rôle. Si le destinataire n’est pas connecté au système, il devra le faire en spécifiant son mot de passe.
1 Le destinataire existe dans le système Karuta et le portfolio est déjà partagé avec le destinataire dans le bon rôle. Si le destinataire n’est pas pas connecté au système, il n’aura pas à le faire : en cliquant sur l’hyperlien, il sera automatiquement dirigé vers la bonne section du portfolio.
2 Le destinataire existe dans le système Karuta mais le portfolio n’est pas déjà partagé avec le destinataire dans le bon rôle. Le système va partager le portfolio dans le bon rôle avec le destinataire. Si le destinataire n’est pas pas connecté au système, il n’aura pas à le faire : en cliquant sur l’hyperlien, il sera automatiquement dirigé vers la bonne section du portfolio.
3 Le destinataire n’existe pas dans le système Karuta. Le système va automatiquement lui créer un compte et lui partager le portfolio dans le bon rôle. Le compte créé est temporaire.
4 Le destinataire qui reçoit le courriel a accès en lecture seulement au contenu de la section du portfolio partagé avec lui. Lors de l’instanciation, le rôle all doit être partagé avec l’utilisateur public.
