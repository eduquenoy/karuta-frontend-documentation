À l'aide de la métadonnée Partager, il possible de permettre aux utilisateurs de partager leur portfolio, au complet ou en partie, avec un tiers et même de donner à ce tiers des droits d'édition sur les éléments partagés. L'utilisateur procède au partage en cliquant sur un bouton, ce qui déclenche l'envoi d'un courriel au tiers avec un hyperlien vers les éléments partagés.

Le partage permet deux possibilités :

  • avertir un acteur du portfolio qu’il doit intervenir ;
  • permettre à un utilisateur qui n’a pas actuellement les droits d’intervenir dans un portfolio de le faire.

Dans le portfolio

Dans l'exemple ci-dessus, lorsque l’étudiant clique sur "Partager avec un pair", le système envoie un courriel au pair avec un hyperlien vers la section du portfolio dans laquelle il pourra inscrire son commentaire.

Il y a plusieurs options pour les conditions de sécurité du partage ; voici quelques cas d’utilisation usuels :

Cas 1 – L’étudiant peut montrer son portfolio à n’importe qui (utilisateur Karuta ou non) en mode lecture seulement (aucun droit d’édition).

etudiant,all,?,4,unlimited,Partager@fr

NOTE 1 : Grâce au point d'interrogation (?), le destinataire n'est pas spécifié à l'avance ; lorsque l’étudiant cliquera sur Partager, une fenêtre s’ouvrira pour lui permettre de spécifier le courriel du destinataire.

Note 2 : Lors de l’instanciation du portfolio, le rôle all doit être partagé avec l’utilisateur public.

Cas 2 – L’étudiant peut montrer son portfolio à n’importe qui (utilisateur Karuta ou non) en mode lecture seulement (aucun droit d’édition) mais le partage a une durée limitée de 24 heures.

etudiant,all,?,4,24,Partager@fr

NOTE 1 : Grâce au point d'interrogation (?), le destinataire n'est pas spécifié à l'avance ; lorsque l’étudiant cliquera sur Partager, une fenêtre s’ouvrira pour lui permettre de spécifier le courriel du destinataire.

NOTE 2 : Lors de l’instanciation du portfolio, le rôle all doit être partagé avec l’utilisateur public.

Cas 3 – L’étudiant demande à son professeur de procéder à son évaluation ; le professeur possède un compte Karuta et le portfolio lui est partagé dans le rôle professeur.

etudiant,professeur,professeur,1,12,Demander une évaluation@fr

Le professeur reçoit un courriel et clique sur l’hyperlien pour procéder à l’évaluation.

Cas 4 – L’étudiant demande à son responsable de stage de compléter son évaluation ; le responsable de stage n’a pas de compte Karuta.

etudiant,evaluateur,yves.lacoste@airfrance.com,3,12,Demander une évaluation@fr

Le responsable de stage reçoit un courriel et clique sur l’hyperlien pour procéder à l’évaluation. Il n’a pas à s’identifier dans le système Karuta. L’hyperlien est valide pendant 12 heures.

Cas 5 – L’étudiant demande à un de ses collègues d’inscrire un commentaire dans son portfolio (évaluation par les pairs) ; le collègue possède un compte Karuta.

etudiant,pair,?,2,12,Demander un commentaire@fr

Le collègue reçoit un courriel et clique sur l’hyperlien pour écrire son commentaire.

NOTE : Grâce au point d'interrogation (?), le destinataire n'est pas spécifié à l'avance ; lorsque l’étudiant cliquera sur Partager, une fenêtre s’ouvrira pour lui permettre de spécifier le courriel du destinataire. On fait ici l’hypothèse que l’étudiant a un grand nombre de collègues et que le designer n’a pas spécifié à l'avance les collègues qui seront mobilisés, de façon à donner plus de latitude à l’étudiant dans le choix du pair qui l’évaluera.

Cas généraux : les deux tableaux ci-dessous expliquent et précisent la valeur des paramètres de la boîte de partage.

Rôle qui partage Habituellement l’étudiant mais tous les utilisateurs qui ont un rôle défini dans le portfolio peuvent aussi partager.
Rôle du destinataire Le rôle qu’aura le destinataire dans le portfolio (par exemple « superviseur » – le destinataire pourra faire ce que peut faire un superviseur).
Rôles et/ou courriels Le rôle du destinataire dans le portfolio ou encore son courriel :
  • Par exemple si « superviseur », le message de partage sera envoyé à la personne qui a le rôle superviseur dans le portfolio. L'utilisateur n'entre pas à la main la personne avec qui partager, ceci est déterminé par le système.
  • On peut aussi inscrire directement une adresse courriel si le destinataire est connu.
  • Finalement on peut remplacer ce paramètre par « ? », ce qui permet à l’utilisateur qui partage de spécifier le courriel de la personne avec qui partager.
Niveau (0,4) Le niveau de sécurité détermine si on oblige le destinataire à se connecter à Karuta avec son identifiant et mot de passe avant de voir ce qui lui est partagé. Il permet aussi à des destinataires qui n’ont pas de compte dans Karuta d’éditer des ressources sans se connecter (niveau 3).
Durée de vie du lien (en heures) Pour des raisons de sécurité, le designer peut spécifier la durée de vie du lien qui sera envoyé.
button label@en/ libellé du bouton@fr Le libellé du menu Partager dans la langue appropriée. Ne pas oublier le / si bilingue.
Condition Par exemple: submitted=='Y' (optionnel).

Niveau Conditions
0 Le destinataire existe dans le système Karuta et le portfolio est déjà partagé avec le destinataire dans le bon rôle. Si le destinataire n’est pas connecté au système, il devra le faire en spécifiant son mot de passe.
1 Le destinataire existe dans le système Karuta et le portfolio est déjà partagé avec le destinataire dans le bon rôle. Si le destinataire n’est pas pas connecté au système, il n’aura pas à le faire : en cliquant sur l’hyperlien, il sera automatiquement dirigé vers la bonne section du portfolio.
2 Le destinataire existe dans le système Karuta mais le portfolio n’est pas déjà partagé avec le destinataire dans le bon rôle. Le système va partager le portfolio dans le bon rôle avec le destinataire. Si le destinataire n’est pas pas connecté au système, il n’aura pas à le faire : en cliquant sur l’hyperlien, il sera automatiquement dirigé vers la bonne section du portfolio.
3 Le destinataire n’existe pas dans le système Karuta. Le système va automatiquement lui créer un compte et lui partager le portfolio dans le bon rôle. Le compte créé est temporaire.
4 Le destinataire qui reçoit le courriel a accès en lecture seulement au contenu de la section du portfolio partagé avec lui. Lors de l’instanciation, le rôle all doit être partagé avec l’utilisateur public.
